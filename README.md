# RICH HALL Minsk

Адаптивный одностраничный сайт банкетного пространства RICH HALL с простой админкой для управления залами.

Банкетное меню вынесено на отдельную страницу `menu.html`. На главной странице оставлен компактный блок с кнопкой перехода к меню.

## Локальный запуск

```bash
python3 -m http.server 4173
```

После запуска сайт доступен по адресу `http://localhost:4173`.

## Управление залами

Админка доступна по адресу `/admin/`. Через неё можно:

- добавлять и удалять залы;
- менять их порядок;
- редактировать название и описание;
- загружать, заменять и удалять фотографии.

Публичные данные хранятся в `content/halls.json`, фотографии — в `assets/halls/`. Админка отправляет изменения в защищённый Cloudflare Worker, который создаёт один коммит в GitHub. После коммита GitHub Pages автоматически обновляет сайт.

Секреты Worker хранятся только в Cloudflare. Они не должны попадать в репозиторий.

## Форма бронирования

Форма отправляет `FormData` методом `POST` в Cloudflare Worker:

```html
<form data-booking-form data-form-endpoint="https://rich-hall-admin-api.n1kseeman-rh.workers.dev/api/booking">
```

Worker валидирует заявку и пересылает её в Telegram. Для работы отправки в Cloudflare должны быть заданы секреты `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`.

Уведомление об успехе появляется только после успешного ответа сервера. Если сервис недоступен или Telegram не настроен, посетителю показывается сообщение с контактным телефоном.

## Публикация

Сайт публикуется через GitHub Pages из корня ветки `main`. Код Cloudflare Worker находится в `worker/`.

Перед финальной публикацией нужно заменить временные изображения и отзывы материалами заказчика, а также указать endpoint формы.
